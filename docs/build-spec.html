01 / Build objective

One engine, two public experiences

IvritCode executes normalized Hebrew source over twenty-two visible base-22 registers plus Aleph Olam. Quantum Etz Chaim presents that same run through services, paths, gates, observation, and manifestation. IvritOS names the orchestration contract between purpose and output.

The register reducer is authoritative. Coherence, graphics, sound, and prose may describe completed state; they may never alter it.
02 / Deliverable boundary

Implemented software, bounded claims

In scope

  • NFC Hebrew input and final-form normalization.
  • Exactly 23 immutable bounded registers.
  • All 22 instructions and a versioned path map.
  • Replayable trace, observation, screen, sound, file, and export.
  • Shared contracts consumed by both sites.

Out of scope

  • Physical qubit control or quantum speed-up.
  • Generated-code execution or runtime network access.
  • Unrecorded permission or global-state mutation.
  • Symbolic language presented as scientific or religious proof.

06 / Cross-site exchange

Reproducible handoff

ivritcode-exchange-0.2 carries source, hashes, initial and final state, engine version, path-map version, seed, complete trace hash, manifestation version, returning letters, and gates.

Quantum Etz Chaim validates the envelope through @qec/spec. Unknown versions, invalid register counts, malformed payloads, and out-of-range values fail closed.
